Overview

The Undergraduate Online Physics Database [UPOD] is an online repository of data designed for students to access physics information, equations, and diagrams to help them in furthering their educations in their university careers. The information on UPOD is added, edited, and removed by moderators and administrators that are approved to do so, where moderator are assumed to be teaching assistants and administrators are assumed to be professors. This report will give a short summary of what was completed, when it was completed, and what remains uncompleted.

Implementation

The implementation of the project was divided into two sections, front-end and back-end, with about half of the members of the group belonging to each. The front-end team was responsible for the design and programming of the user interface using HTML, CSS and JavaScript. The back-end team was involved with storing data in a database using JDBC, and manipulating data using Java.

Completion

The project was not completed as originally intended, although many features were found to be working and functional to the degree that they were specified. The front-end HTML pages are complete, while the underlying JavaScript used to make these pages interactive was close to completion [as seen on the edit page]. The JSON was also close to demonstrative functionality. The back-end were able to build and complete a majority of their components (such as the search, page storage, retrieval and deletion) but not enough to fully interact with the front end. Essentially a large number of working or almost working parts, although we were unable to thoroughly consolidate these parts into one.

Challenges We Faced

We had trouble getting all the group members together with our varying schedules. There were also issues with getting the JSON to communicate with the data in the back-end. There were also issues with getting the front-end HTML working with all browsers, for example, it seemed like FireFox rendered the top search bar beside the body rather than above it, on every page that had it in the header.

Future Improvement

As the project went on, certain individuals emerged as being more experienced with certain aspects [such as HTML and JavaScript]. It might have been helpful to assign these people as leaders earlier on in the development processes so they could better express just exactly how we would do things, as opposed than the laissez-faire style employed in the early stages.
